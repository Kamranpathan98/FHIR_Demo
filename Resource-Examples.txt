Why is Encounter a separate resource in FHIR, instead of just being part of Patient like in HL7 v2?
What advantage does this give in real systems?

=> Encounter is separate resource because it contains all history of events occuring for patient. This make 
it easier so if a doctor needs only the events/medications of patient there no need to fetch whole patient data from birth

Strong Interview Version

Encounter is modeled as a separate resource in FHIR because a patient can have multiple, 
independent clinical interactions over time. By separating Encounter from Patient, 
FHIR allows each visit to have its own context, timeline, participants, location, and linked clinical data. 
This enables precise querying, auditing, and reconstruction of clinical workflows without overloading the Patient resource.

Why is it important that Observation links to both Patient and Encounter?
What problem would you face if Encounter was missing?

We need to link Observation to both Patient and Encounter because linking Patient doctor will understand
the test are done for which patient and linking ecounter will actually provide description for what patient was at hospital for


Observation links to Patient to identify who the data belongs to, and links to Encounter to capture the clinical context in which the data was collected. 
Without Encounter, we lose the ability to reconstruct visit-specific workflows, timelines, and clinical reasoning. 
This makes auditing, CDS, and longitudinal analysis unreliable.

=> Keeping Questionare and QuestionnaireResponse separate make it very flexible and reusable. Same questionnaire can be used
across the system for every patient, and separate response help in differentiate each patient response

FHIR separates Questionnaire and QuestionnaireResponse to decouple form definition from form data. 
This allows the same questionnaire to be reused across patients, encounters, and systems, while each response 
remains patient- and context-specific. This design enables interoperability, versioning, and workflow automation.
